Troubleshooting Apple Silicon

Issue 1: USB Drive Not Detected

If your bootable USB doesn’t appear at startup, ensure it was created properly using Balena Etcher. Re-flash the ISO and verify that you’re using the ARM64 image.

Tips:

  • Use USB 3.0 drives for better compatibility
  • Try plugging the USB directly into your Mac (avoid hubs)
  • Hold the Power button until “Options” shows, then open “Startup Security Utility” and set to Allow booting from external media

Issue 2: Installer Freezes or Reboots

This can be caused by insufficient RAM or faulty storage media. Ensure the ISO isn't corrupted.

sha256sum LNP-OS-ARM64-PC.iso

Compare the hash to the one on the download.html page. Use a reliable USB stick (SanDisk/Kingston recommended).

Issue 3: Wi-Fi, Bluetooth, or Audio Not Working

LNP OS uses a hybrid auto-driver system. To manually trigger detection, open the terminal and run:

sudo lnp-device-manager --scan

Alternatively, go to Device Manager from the system menu. It will automatically search and install available drivers if a network connection is active.

Issue 4: "Install Alongside macOS" Fails

Make sure macOS is installed on a GPT/APFS volume. Encrypted volumes or FileVault may block partitioning.

diskutil apfs list

If errors appear, use the Manual Installation mode and partition with:

sudo cfdisk /dev/nvme0n1

Issue 5: Can't Boot LNP OS After Installation

If you’re dual-booting and don’t see LNP OS in startup options:

  • Shut down completely
  • Hold Power until “Options” menu appears
  • Select LNP OS from the list
  • Use Startup Disk in macOS to set default if needed
